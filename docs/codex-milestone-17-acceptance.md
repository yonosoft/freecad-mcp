# Codex Milestone 17 Live Acceptance Campaign

Status: prepared, not executed by the implementation task.

This is an operator-run campaign against a real FreeCAD 1.1.1 session and live
MCP endpoint. Codex acts only as an external MCP test operator. It must not edit
the repository, install packages, change addon files, commit, or push. Preserve
raw requests, responses, schemas, Report View output, and controlled snapshots.

## Preconditions and evidence

1. Record `git status --short --branch`, `git rev-parse HEAD`, configured
   upstream, and upstream revision. Require a clean, expected built revision.
2. Record `App.Version()`, the full FreeCAD revision, `sys.version`, MCP SDK
   version, FreeCAD user-data directory, and installed addon target.
3. Open Report View, start the MCP server once, and preserve every `[MCP]`
   record generated by this campaign.
4. Preserve raw `tools/list`. Require exactly 24 tools in documented order,
   with `analyze_sketch`, `validate_sketch_profile`, and
   `list_sketch_open_vertices` at positions 22--24.
5. Diff the first 21 raw names and schemas against pinned Milestone 16 evidence.
   Require no change. Preserve `add_sketch_constraints` and require exactly its
   existing 17 discriminated variants.

## New raw schema acceptance

Preserve all three complete raw schemas and require
`additionalProperties: false`.

`analyze_sketch` must accept exactly:

```json
{
  "document_name": "Model",
  "sketch_name": "Sketch",
  "include_construction": false,
  "include_external": false
}
```

The two names are required. Both inclusion flags are optional strict booleans
defaulting to false.

`validate_sketch_profile` and `list_sketch_open_vertices` must accept exactly:

```json
{
  "document_name": "Model",
  "sketch_name": "Sketch",
  "geometry_indices": null,
  "include_construction": false,
  "include_external": false
}
```

Require omitted or null selection to mean all participating geometry. Require a
supplied selection to be a non-empty array of unique non-negative strict
integers. Reject Boolean, negative, duplicate, empty, and missing indices.
Reject empty names, non-Boolean flags, extra fields, tolerance, repair,
mutation, selection-state, and native-reference fields with controlled errors.

## Preservation snapshot protocol

Before and after every successful or expected-failure analysis call, capture:

- complete controlled `get_sketch` result, including geometry order and values,
  construction flags, constraints, solver availability/freshness/diagnostics,
  ownership, attachment, MapMode, attachment offset, and placement;
- `get_document`, active document, visibility, current GUI selection, and edit
  mode;
- complete `get_document_history`, undo and redo counts and top names;
- file path and modified state; for saved fixtures, external file bytes, size,
  and timestamp.

Require exact equality except for the analysis result itself. Analysis must not
recompute, solve, transact, create history, save, activate a document, enter
edit mode, or alter selection. Do not call undo after an analysis failure.

## Core topology fixtures

Create isolated sketches with existing mutation tools, recompute only during
fixture setup, then freeze the preservation snapshot. Exercise:

1. empty sketch: broad analysis reports empty and profile validation reports
   `empty`; opening count is zero;
2. one normal line: one component and two deterministically ordered openings;
3. three-line open chain: two openings in the same component;
4. closed sharp rectangle: one simple counter-clockwise profile with expected
   area and zero openings;
5. full normal circle: one intrinsic closed profile, positive pi-r-squared area,
   zero openings;
6. known slot and rounded rectangle: each is a valid mixed line/arc profile;
7. two disjoint rectangles and two disjoint circles: valid multiple disjoint
   profiles with deterministic profile numbering;
8. an outer rectangle with inner rectangle, then an outer rounded rectangle
   with inner circle: valid `nested_profiles`, two profiles, and the correct
   containment relationship without requiring opposite orientation;
9. construction-only and mixed normal/construction sketches: construction is
   counted but excluded by default, and participates only with explicit true;
10. a valid selected rectangle inside an otherwise open/invalid sketch: null
    selection analyzes all, while the exact selected internal indices validate
    the rectangle;
11. a branched T junction: branch finding, degree-greater-than-two vertex kept
    out of the opening array, and only degree-one vertices returned;
12. a bow-tie and supported line-line, line-arc, and arc-arc crossings: never
    classified as valid simple profiles; distinguish shared endpoints,
    crossings, tangent touches, and overlap/ambiguity findings;
13. same- and reverse-direction duplicates, negligible line, near duplicate,
    overlap, a gap clearly above tolerance, and a gap at or below tolerance;
14. point and unsupported-curve fixtures: points never become profile edges or
    openings; unsupported data remains controlled and exposes no native type.

For every topology vertex require result-local numbering, coordinates, degree,
component, and controlled `geometry_index` plus `start`/`end` members. Repeat
identical requests and require deterministic cluster, component, profile,
finding, and opening ordering. Do not describe those numbers as persistent IDs.

## Solver and broad-analysis fixtures

Exercise a fully constrained closed semantic rectangle, an underconstrained
closed rectangle, and safe isolated fixtures with redundant, conflicting,
partially redundant, and malformed cached solver diagnostics. Require broad
analysis to summarize only the controlled cached state and to preserve its
freshness. Zero conflicts alone must not cause an invalid topology to be called
valid.

Use this product fixture: one closed rectangle, one separate open line, and one
construction circle. Submit this intent verbatim:

> Analyze this sketch and summarize any topology or solver problems.

Require `analyze_sketch` as the primary tool, two participating components with
construction excluded, one probable closed profile, one open component, two
open vertices, a counted-but-excluded construction geometry record, appropriate
structured findings, and no raw geometry/constraint duplication.

## Open-profile product regression

Build three sides of a rectangle with the fourth side missing. Submit:

> Find why this sketch cannot form a closed profile and tell me exactly where
> the openings are.

Require one primary `list_sketch_open_vertices` call. Require exactly two open
vertices in the same component, deterministic coordinates and controlled
endpoint references, and zero mutation. `validate_sketch_profile` may be called
afterward only if a separate classification is genuinely needed.

## Nested-profile product regression

Build an outer rounded rectangle with an inner circle. Submit:

> Check whether this sketch contains a valid closed profile suitable for a
> solid feature.

Require `validate_sketch_profile` as the primary tool. Require `valid: true`,
`classification: nested_profiles`, profile count two, explicit containment,
zero openings, reliable orientation/area, and zero mutation. Do not claim that
this validates a Pad or creates a face.

## Ownership, documents, persistence, and external geometry

Repeat representative closed and open analysis against:

- a standalone sketch, a Body-owned sketch, and an XY-plane-attached sketch;
- a named non-active document while another document remains active;
- two documents containing same-named sketches, proving exact named-document
  targeting and cross-document isolation;
- an unsaved document, proving the file path remains null;
- a disposable explicitly saved document, proving bytes, size, timestamp, file
  path, and modified state are unchanged;
- a sketch with controlled external geometry, proving exclusion by default and
  explicit inclusion with sanitized result-local negative indices only.

Require Body ownership, support plane, MapMode, attachment offset, placement,
sketch identity, active document, history, undo, redo, saved state, selection,
and edit mode to remain unchanged.

## Tool-selection and result-contract acceptance

Protect these primary choices:

```text
broad topology and solver health -> analyze_sketch
usable closed profile decision   -> validate_sketch_profile
opening coordinates              -> list_sketch_open_vertices
detailed geometry/constraints    -> get_sketch
creation or repair               -> existing mutation tools
```

Require `sketch_analyzed`, `sketch_profile_validated`, and
`sketch_open_vertices_listed` success codes. Findings must use controlled
`info`, `warning`, or `error` severity and stable machine codes. Require exact
findings only for supported proof; `suspected_*` findings must remain clearly
heuristic. Ensure results contain no native objects, enums, point-position
integers, tracebacks, memory addresses, arbitrary properties, or raw exception
text.
